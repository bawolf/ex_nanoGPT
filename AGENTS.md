# AGENTS.md

## Project Overview

Elixir port of nanoGPT/nanochat. Two model implementations (`v1_compact.ex`, `v2_compact.ex`) plus exploded module-by-module versions for teaching via Livebook notebooks.

## Dependencies

- **EMLX** is Apple Silicon only. It's conditionally included in `mix.exs` (darwin only). On Linux, only EXLA is available.
- Set `NX_BACKEND` env var to choose backend: `emlx` (default, macOS GPU), `emlx_cpu`, or `exla`. CI uses `exla` with `XLA_TARGET=cpu`.

## Tests

Run all tests: `mix test`

### Test Tags

Tests that can't run in CI are tagged and excluded:

- `@tag :golden` — Tests comparing Elixir output against PyTorch golden `.npy` files. These files are **gitignored** (`test/golden/*.npy` and `test/support/golden_v2/`), so they only exist locally after running the Python extraction scripts.
- `@tag :external` — Tests that download data from the internet (e.g. Shakespeare dataset).

CI runs: `mix test --exclude golden --exclude external`

When adding new tests:
- If a test loads from `test/golden/` or `test/support/golden_v2/`, tag it with `@tag :golden`.
- If a test requires network access, tag it with `@tag :external`.
- Use `@moduletag` when all tests in a module share the same tag.

### Golden Data

Golden `.npy` files are generated by Python scripts that run the reference PyTorch implementations and save intermediate tensors. They are not checked in. See `test/golden/extract.py` for v1.

## Code Style

- `mix format` is enforced in CI (uses `.formatter.exs` config).
- Compact model files (`v1_compact.ex`, `v2_compact.ex`) are intentionally dense single-file implementations. The exploded modules in `lib/ex_nano_gpt/` and `lib/ex_nano_gpt/v2/` are the same code split for readability.
- Notebooks in `notebooks/` are Livebook `.livemd` files. They install their own deps via `Mix.install` and are self-contained.

## CI

GitHub Actions workflow at `.github/workflows/ci.yml`:
- Compile with `--warnings-as-errors`
- Format check
- Tests (excluding golden and external)
- Unused dependency check
